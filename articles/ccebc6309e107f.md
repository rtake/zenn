## 構想

- Agent Serviceのアクティビティログを監視するアラートルールを設定する
- エージェント発行イベントのログからエージェントの情報をパースするAzure Functionsを作成する
- エージェント情報を受信し、エージェント定義ファイルをGitHubにアップロードする
  - エージェント定義を受信してGitHubのブランチにPushするワークフローを実装する
  - 受信したエージェント情報をもとにAgent Serviceからエージェント定義をエクスポートし、ワークフローを起動する
- 2つの関数をQueueストレージを介して統合する
  - 一方の関数はQueueに実行結果を出力するように、他方の関数はQueueに対する項目追加をトリガーとして起動できるように設定する
- (別記事) PRマージ時にAgent ServiceにデプロイするワークフローをGitHub Actions上に構築する

## 処理概要

1. Agent Serviceでエージェントを発行する
2. Azure Monitorがエージェント発行イベントを検知しアラートを発報する
3. アラートのアクショングループに設定した監視用Azure Functionsが起動し、イベントログをパースしてエージェントの情報を取得しQueueに格納する
4. Queueへのアイテム追加をトリガーとして、エージェント定義をエクスポートしGitHubへとアップロードするAzure Functionsが起動する
   - エージェント定義をブランチにコミットし、Push・PR作成を実行するGitHub ActionsワークフローをAzure Functionsから起動する
   - PRマージ時に別のAgent Serviceにエージェントを自動デプロイする
